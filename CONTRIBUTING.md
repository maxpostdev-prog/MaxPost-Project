# Contributing to MaxPost

## Core rules

1. One tool solves one primary task.
2. Documentation precedes implementation.
3. Business logic belongs in `maxpost-core`.
4. Presentation belongs in `maxpost-theme`.
5. New UI components must follow the design system.
6. User trust is never sacrificed for revenue.

## Workflow

1. Open or select an issue.
2. Create a focused branch.
3. Keep changes small and reviewable.
4. Add or update documentation.
5. Test the change.
6. Open a pull request with a clear summary.

## Commit style

Use concise conventional prefixes:

- `feat:` new functionality
- `fix:` bug fix
- `docs:` documentation
- `refactor:` internal code improvement
- `test:` tests
- `chore:` maintenance

## Pull requests

A pull request should include:

- problem statement
- implemented solution
- files or modules affected
- test evidence
- screenshots for visual changes
- migration notes when relevant
