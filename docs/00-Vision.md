# MaxPost Vision

**Version:** 1.0 Alpha  
**Status:** Living Document

> Software should solve problems, not create them.

## Mission

Create small, reliable applications that remove repetitive work from a user's day.

## Product principle

> One Tool. One Task. Done.

Each application exists to perform one primary job exceptionally well. If a new feature creates a separate workflow, it should become a separate application.

## Long-term vision

MaxPost should become a trusted destination for lightweight Windows utilities. Users should remember MaxPost because the software is predictable, safe and useful—not because the brand demands attention.

## Core principles

### Simplicity first

Remove unnecessary settings, dialogs and steps.

### Speed is a feature

Applications should launch quickly, remain responsive and avoid unnecessary background work.

### Clarity over cleverness

A basic task should not require documentation.

### Trust is everything

MaxPost will not use fake download buttons, bundled installers, hidden telemetry or deceptive advertising.

### Respect the user

Respect the user's time, privacy, attention, hardware and choice.

### Consistency

Learning one MaxPost application should make every other MaxPost application familiar.

## User promise

Every application should be:

- lightweight
- fast
- safe
- reliable
- easy to understand
- easy to remove
- free whenever the business model permits

## Business philosophy

Revenue should come from value rather than frustration.

Acceptable:

- donations
- premium applications
- business licences
- clearly marked sponsorships
- relevant affiliate partnerships
- internal recommendations

Forbidden:

- fake download buttons
- forced waiting
- hidden installers
- misleading advertising
- selling user data

## Success criterion

The project succeeds when users say:

> When I need a small Windows utility, I check MaxPost first.
